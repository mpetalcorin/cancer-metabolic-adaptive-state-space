# Datasheet for Datasets## Dataset nameSimulated Cancer Metabolic-Adaptive State Space Datasets## MotivationThese datasets were created to support a proof-of-concept framework in which cancer is modeled as a **dynamic metabolic-adaptive state space** rather than a static disease class.The datasets were designed to answer questions such as:- Can tumour populations be represented as structured adaptive states?- Can metabolism, DNA repair, and plasticity be integrated in one simulated landscape?- Can machine learning recover those states and transitions?- Can adaptive bottlenecks be identified under therapy pressure?The datasets were created for research, prototyping, and systems-level hypothesis generation.## CompositionThe dataset collection includes two main parts:### 1. Cross-sectional datasetA simulated dataset containing tumour samples across:- 6 adaptive tumour states- 5 stress conditions### 2. Longitudinal datasetA simulated lineage-style dataset containing:- repeated time points- stress-driven transitions between states- multiple stress regimes## States representedThe six adaptive states are:- Glycolytic_Proliferative- OXPHOS_Competent- Glutamine_Addicted- Hybrid_Plastic- Repair_Deficient_Stressed- Drug_Tolerant_AdaptiveThese states are intended to represent conceptual regions of a metabolic-adaptive landscape.## Features includedThe datasets include the following main variables:- glucose_uptake- lactate_secretion- OCR- ECAR- glutamine_uptake- LDHA_expression- GLS_expression- redox_ratio_NADPH_GSH- ROS_stress- ATP_pool- DNA_repair_capacity- replication_stress- hypoxia_response- EMT_plasticity- stemness- proliferation- drug_tolerance- mt_healthAdditional variables include:- state- stressor- vulnerability_score- lineage_id- timepoint## Instances### Cross-sectional datasetEach row represents one simulated tumour sample under a given state and stress condition.### Longitudinal datasetEach row represents one simulated lineage-state observation at a specific time point.## Data generation processThe datasets were generated by sampling from multivariate normal distributions centered on **state-specific mean feature profiles** with controlled overlap introduced through shared standard deviations.Stress conditions were modeled as directional perturbations on state means.Longitudinal transitions were generated using **rule-based transition probability maps** designed to reflect plausible stress-driven adaptive movement, especially toward therapy-tolerant states under treatment pressure.## Benchmarking logicThe datasets were benchmarked to **literature-supported qualitative patterns**, including:- glycolytic and lactate-high proliferative behaviour- OXPHOS-competent metabolic behaviour- glutamine dependence and GLS-linked adaptation- repair deficiency with replication and oxidative stress- plastic, stem-like, therapy-tolerant adaptive behaviourThese benchmarks are conceptual and biological, not numerical clinical calibration targets.## Source dataThe dataset is **synthetic**.It was not collected from:- patients- animals- clinical records- public omics repositories- laboratory experimentsNo real human subject data are included.## Recommended usesThese datasets are appropriate for:- systems biology prototyping- machine-learning method development- simulation-based hypothesis generation- educational demonstration- state-space and transition analysis- conceptual cancer framework testing## Not recommended usesThese datasets should not be used for:- clinical interpretation- medical advice- diagnostic algorithm deployment- biomarker benchmarking against patient cohorts- claims of real-world tumour prevalence- treatment selection## PreprocessingThe raw simulated values are already numeric and bounded.Downstream preprocessing in the project includes:- train-test splitting- standardization for selected models- dimensionality reduction- clustering- permutation-importance analysis- transition-matrix estimation## LabelingState labels are assigned directly by the simulation engine.They are not noisy crowd labels and are not experimentally adjudicated labels.Because labels are generated from the simulation design itself, model performance on these labels should be interpreted as **proof-of-concept recovery of designed structure**, not external validation.## DistributionThe dataset is intended to be distributed as CSV files in the repository output directory.Suggested files include:- `simulated_cancer_state_space_cross_sectional.csv`- `simulated_cancer_state_space_longitudinal.csv`- `state_summary_means.csv`- `stressor_summary_means.csv`- `transition_matrix_therapy.csv`- `transition_matrix_nutrient.csv`- `transition_matrix_hypoxia.csv`- `transition_matrix_genotoxic.csv`- `transition_bottleneck_scores.csv`## Known limitationsKey limitations include:1. The data are simulated, not experimentally measured.2. Numeric values are not calibrated to one real cohort.3. Feature covariance is simplified relative to real tumour biology.4. Tumour microenvironment complexity is reduced.5. Immune interactions are absent.6. Spatial heterogeneity is not directly modeled.7. Some important metabolic axes are simplified or omitted.8. Transition rules are hand-designed, not learned from longitudinal real-world observations.## BiasesBecause the data are simulation-driven, the main biases arise from:- the chosen state definitions,- the selected features,- the manually encoded stress effects,- the hand-specified transition probabilities,- the assumption that literature-derived qualitative patterns can be represented in this simplified feature space.These biases should be treated as part of the conceptual design of the proof of concept.## Sensitivity and privacyThe dataset contains **no personal data** and no protected health information.Privacy risk: **none from source data**Sensitivity risk remains at the level of possible **overinterpretation**, not confidentiality.## Maintenance and future expansionFuture versions may include:- real multi-omics grounding- tumour-type-specific parameterizations- spatial structure- immune-metabolic interaction layers- more explicit therapy modules- experimentally inferred transitions- expanded pathway coverage## Dataset summary statementThis dataset collection is best understood as a **simulation-based systems oncology resource** designed to test whether cancer can be modeled as a metabolic-adaptive state space with biologically interpretable states, transitions, and bottlenecks.
